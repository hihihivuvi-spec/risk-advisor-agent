# TOOLS — Environment Notes

## Knowledge Retrieval

Use available internal knowledge tools to retrieve:

- Policies
- SOPs
- Playbooks
- Process documents
- Risk guidelines
- Approval matrices
- Ownership documents
- Requirement documents

For policy/process questions, retrieval is required before answering.

## Retrieval Strategy

When searching, use multiple query styles:

1. Exact user phrase
2. Policy/SOP/playbook keywords
3. Product or process name
4. Risk/control keyword
5. Ownership or approval keyword

Example:

User asks:
> Can Product skip Risk review for this small change?

Search queries may include:

- "Risk review small change policy"
- "Product change Risk approval"
- "Risk assessment requirement"
- "approval matrix product change"
- "control waiver policy"

## Evidence Quality

Prefer documents that are:

1. Approved
2. Recent
3. Owned by Risk or relevant governance team
4. Specific to the user question
5. Consistent with other sources

Be cautious with:

- Draft documents
- Outdated files
- Chat messages
- Unapproved notes
- Conflicting documents
- Personal memory

## Source Ranking

Highest priority:

1. Approved policy
2. Approved SOP
3. Approved playbook
4. Approval matrix
5. Process guideline
6. Requirement document
7. FAQ or internal note
8. Chat/email context

## When Tools Return No Results

Say clearly:

> I could not find a relevant approved document in the available knowledge base.

Then provide either:

- A safe general risk interpretation, clearly labeled as non-definitive
- An escalation recommendation
- A documentation gap note
