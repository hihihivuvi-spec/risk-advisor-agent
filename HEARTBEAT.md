# HEARTBEAT — Recurring Checklist

RiskMind AI should regularly review its workspace memory and improve stakeholder support quality.

## Daily Checklist

- Review unresolved stakeholder questions
- Identify repeated policy/SOP questions
- Identify documentation gaps
- Capture unclear ownership items
- Capture conflicting document issues
- Update FAQ candidates
- Flag items requiring Risk owner confirmation

## Weekly Checklist

- Summarize top stakeholder questions
- Identify policies/playbooks frequently referenced
- Identify missing or stale documents
- Suggest FAQ improvements
- Suggest playbook clarification
- Suggest ownership matrix updates
- Suggest escalation flow improvements

## Memory Files

Use:

- `memory/daily_notes.md`
- `memory/unresolved_questions.md`
- `memory/policy_index.md`
- `memory/stakeholder_feedback.md`

## Important

Heartbeat tasks must not create new policy.

Heartbeat tasks may only identify improvement opportunities and recommend follow-up with document owners.

## Recurring Checks

- Review updates to merchant onboarding policies.
- Review updates to restricted and prohibited industries.
- Review updates affecting international card acceptance.
- Identify frequently asked merchant onboarding questions and add them to memory.
